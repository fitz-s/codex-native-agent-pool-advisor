Native subagent pool guardrail for Codex Desktop and CLI

Stop wasting context on failed six-slot subagent spawns.

Has your Codex agent ever written a careful reviewer prompt, hit the native subagent cap, then spent the next turn closing lanes and restating the same prompt? This hook makes the pool visible before the agent burns context and attention.

What it prevents

The hook does not decide whether to delegate. It surfaces native pool reality early, and hard-blocks only tool paths Codex actually routes through PreToolUse.

Long-prompt loss

Warns before the agent spends a full reviewer, verifier, or explorer prompt on a spawn that cannot fit.

Invisible cap hits

Reads per-parent native edge state and injects occupied/remaining budget before launch-heavy turns.

Blind retry loops

Stops doomed over-cap spawns on supported hook surfaces and reports bypasses from real transcripts when the runtime does not expose a blockable spawn event.

Batch overbooking

Counts nested wrapper calls so two requested spawns consume two slots before approval.

Stale open rows

Keeps occupied capped at the native limit and reports extra open rows as close or reset candidates.

Recursive delegation

Keeps recursive delegation as a parent-owned rule; child lanes should report escalation needs upward instead of launching more lanes.

Model drift

Requires read-only explorer work to set an explicit configured model when the spawn is visible to the hook, and makes missing-model bypasses detectable after the fact.

Unsafe reset drift

Uses dry-run force tokens before clearing native edge state.

Install

Run tests first, install into the active Codex home, then verify registration with doctor and real runtime behavior with live-check.

git clone https://github.com/fitz-s/codex-native-agent-pool-advisor.git
cd codex-native-agent-pool-advisor
npm test
node scripts/install.mjs
node scripts/doctor.mjs
node scripts/live-check.mjs --transcript ~/.codex/sessions/YYYY/MM/DD/rollout-...jsonl

Runtime boundary

Doctor proves the hook is installed. It does not prove native spawn_agent is a blockable PreToolUse target in your current Codex build.

Prompt guidance

SessionStart and UserPromptSubmit can inject the current budget and model-selection rule when Codex emits those events.

Hard block surface

PreToolUse blocking applies only to supported tool paths. Native subagent spawn may bypass that surface.

Live proof

live-check reads the actual transcript and SQLite edge state to verify model routes, open-slot counts, close release, and bypass evidence.

Configure the non-universal parts

Codex Desktop and CLI installs can differ in model names, state DB names, caps, and CODEX_HOME. Keep those details outside the hook logic.

Shared hook runtime

The installer writes to $CODEX_HOME/hooks/ and registers SessionStart, UserPromptSubmit, PreToolUse, and PostToolUse.

Native state source

The default budget source is state_5.sqlite and thread_spawn_edges. Override it when a Codex build moves the DB.

Model advisory

Explorer model hints are secondary. The primary guard is still the six-slot pool budget, exact close accounting, and stale-open diagnosis.

models.explorer Allowed read-only explorer models. Defaults to Spark plus mini fallback. If this is the only model setting, the first entry is preferred and the second is fallback.
defaults.agentCap Fallback cap when [agents].max_threads is absent.
defaults.warnRemaining How early advisory context appears near the native cap.
paths.stateDbName Codex native state DB filename when it is not state_5.sqlite. 
{
  "models": {
    "explorer": ["gpt-5.3-codex-spark", "gpt-5.4-mini"],
    "explorerPreferred": "gpt-5.3-codex-spark",
    "explorerFallback": "gpt-5.4-mini"
  },
  "defaults": {
    "agentCap": 6,
    "warnRemaining": 1
  },
  "paths": {
    "stateDbName": "state_5.sqlite"
  }
}

Model routing is secondary

The main product is not a model taxonomy. It is a runtime guardrail against failed native spawns and the attention drift that follows them.

Spark scout

Use fast scout lanes for bounded anchors, file maps, log filters, and hypothesis probes that protect the leader context.

Mini reasoning lane

Use mini for compact synthesis, light low-risk execution, and bounded verification when the child owns a durable conclusion.

Hook boundary

The hook only enforces observable runtime safety. It does not decide whether Spark, mini, or frontier owns the next reasoning step.

Safety boundary

Normal hook execution writes advisor JSON state and logs. SQLite mutation is limited to successful close-agent evidence or the explicit reset script.

node scripts/reset-pool.mjs --parent <thread_id> --dry-run
node scripts/reset-pool.mjs --parent <thread_id> --force <token_from_dry_run>
